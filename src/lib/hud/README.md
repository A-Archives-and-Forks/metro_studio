# lib/hud 目录说明

- `renderModel.js`
  - 从工程数据中提取指定线路的主路径，并自动生成方向选项（常规线路为双向“开往终点”，环线为正/反向）。
  - 对线路拓扑执行主通路求解：
    - 自动过滤无效边与孤立分量
    - 非环线优先按端点最远距离求主干路径
    - 环线在满足简单环条件时按环顺序生成全站序列
  - 生成车辆 HUD 渲染模型：
    - 站序、线条主路径、终点方向文本
    - 换乘标识（圆形换乘图标 + 倒三角标 + `换乘/Transfer` + 多线路牌；线路牌直接显示真实线路名，如 `2号线`、`济阳线`、`云巴1号线`，仅展示非当前线路；三角标颜色跟随首个换乘线路；线路牌矩形采用放大尺寸；且始终位于站名文字的另一侧）
    - 上侧换乘块自动抬高线路牌并拉开 `换乘/Transfer` 间距，避免英文被线路牌遮挡
    - 换乘线路识别使用统一线路 ID 键（兼容字符串/数字 ID），避免本线被误显示为换乘线
    - 站名标签使用水平排版（角度 `0`），并保留离线偏移避免压线（如 `山东职业学院 / Shandong Vocational College`）
    - 段间方向箭头（双箭头标记）
    - 超长线路单次折返（约 `15` 站阈值，两排 + 右侧单弯连接，且上下排间距按换乘块高度自适应增大，避免上下换乘标识冲突）
    - 未折返时固定 HUD 长度，避免随站数变化导致版式抖动
